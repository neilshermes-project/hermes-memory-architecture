# Contributing to Hermes Memory Architecture

Thank you for your interest in improving the Hermes memory system.

## Ways to Contribute

- Improve the replication scripts
- Add support for new embedding models
- Enhance the backout / promotion process
- Report issues with specific Hermes versions
- Submit integration proposals for the core Hermes project

## Development Setup

1. Clone the repo
2. Run `python replicate.py --auto-snapshot` to test locally
3. Always develop inside a snapshot — never against production

## Pull Request Guidelines

- Keep changes focused
- Update documentation when behavior changes
- Ensure `replicate.py` still works after modifications
- Add tests where feasible

## Code of Conduct

Be respectful. This project aims to make Hermes safer and more capable for everyone.
